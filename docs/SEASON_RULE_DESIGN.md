# 赛季规则设计范式

> 本文档定义赛季积分榜规则展示的设计规范和约束。所有赛季规则组件（S1-S5）必须遵循。

---

## 1. 布局顺序

规则展示必须遵循「因果链」顺序：

```
① 规则触发条件 → ② 规则结果 → ③ 衍生状态 → ④ 排名
```

示例（S5）：

```
┌─────────────────────┐
│ ① 轮次骰子          │  ← 每轮第一件事：掷骰子
│ R1 ⚂ 秩序          │
│ R2 ⚄ 秩序          │
│ R3 ⚀ 异变          │
└─────────────────────┘
┌──────────┐┌──────────┐
│ ② 秩序   ││ ② 异变   │  ← 骰子决定的结果
│ 抵抗·爆发││ 混沌·债务│
└──────────┘└──────────┘
┌─────────────────────┐
│ ③ 技能卡            │  ← 规则产生的道具/状态
│ 暂停卡 · 债务卡      │
└─────────────────────┘
┌─────────────────────┐
│ ④ 排名              │  ← 最终结果
└─────────────────────┘
```

## 2. 规则展示原则

### 2.1 赛季名不重复

- 赛季名仅出现在 `SeasonTabs` 标签中。
- 规则卡片内**禁止**出现 `S5-异变秩序` 等与 Tab 重复的标题。
- 卡片标题用功能性信息（如 `第 3/9 轮`、`轮次骰子`）。

### 2.2 规则信息分层

| 层级 | 展示位置 | 内容 |
|------|---------|------|
| 一眼可见 | 卡片副标题 / chip 标签 | Buff 关键词（如「抵抗·爆发·贯穿」） |
| 点击查看 | 点击卡片 → Sheet 弹窗 | Buff 的完整触发条件和效果 |
| 代码参考 | 组件内注释 / `rules/` 目录 | 精确的实现逻辑 |

### 2.3 规则 Sheet 样式

- 底色：`bg-surface-hover` + `border border-line-light`（中性色，不用红/绿色）
- 每个 Buff 一个独立色块，按 ①②③ 编号
- 标题用 `text-fg`，描述用 `text-fg-muted`

### 2.4 交互一致性

- **所有规则入口统一为直接点击**：点击卡片本身或关键元素，不使用独立的 `ℹ️` 按钮。
- 技能卡上的「使用」按钮是游戏操作，不是规则入口，两者需区分。

### 2.5 视觉编码

- 模式色彩仅用于**游戏玩法元素**（骰子芯片、模式卡片、技能卡边框），不用于规则说明文字。
- 过去/未来/今天等时间状态用透明度区分（参考日历组件）。

## 3. 组件约定

### 3.1 文件命名

```
client/src/components/season/S{n}Rankings.vue
```

每个赛季规则一个组件，由 `RankingsHubView.vue` 按 `ruleId` 动态加载。

### 3.2 Props 约定

```js
defineProps({
  rankings: Array,   // 排名数据（必传）
  season: Object,    // 当前赛季对象
  rounds: Array,     // 轮次列表
  matches: Array     // 比赛列表（S4 专用）
})
```

### 3.3 规则文字来源

- 规则描述从 `client/src/rules/s{n}.js` 中的注释和常量推导
- 不凭空编造规则文字
- 修改规则时需同步更新规则组件的展示文字

## 4. 设计约束

### 4.1 不做

| 禁止 | 原因 |
|------|------|
| 规则卡片标题与 SeasonTabs 重复 | 信息冗余 |
| 规则 Sheet 用红/绿等游戏色 | 规则是中性说明，不是游戏元素 |
| 大段纯文本堆叠 | 不可扫读，移动端体验差 |
| 自定义 CSS 类名（如 `eq-*`） | 必须用项目 Tailwind tokens |
| 独立的 ℹ️ 按钮作为唯一规则入口 | 交互不一致 |
| 盒套盒布局（`bg-surface` 内嵌 `bg-surface`） | 违反 Apple 原生扁平设计 |

### 4.2 必须

| 必须 | 原因 |
|------|------|
| 组件内规则文案与 `rules/` 代码逻辑一致 | 避免 UI 描述与实际机制不符 |
| 点击元素有 `active:scale-95` 反馈 | 触控友好 |
| 空状态用 `EmptyState` 组件 | 统一视觉 |
| 分制/分数用 `font-display` + `font-mono` | 数字可读性 |

## 5. 新增赛季规则检查清单

新增 S{n} 规则时，逐项检查：

- [ ] `rules/s{n}.js` 完成规则逻辑
- [ ] `S{n}Rankings.vue` 组件创建
- [ ] 布局顺序遵循因果链（触发 → 结果 → 状态 → 排名）
- [ ] 赛季名不重复（不在卡片标题中再次出现）
- [ ] 规则关键词出现在卡片副标题（一眼可见）
- [ ] 完整规则可通过点击查看（Sheet 弹窗）
- [ ] 规则 Sheet 使用中性色（`bg-surface-hover` + border）
- [ ] 自定义 CSS 不超过必要范围，优先用 Tailwind tokens
- [ ] 无盒套盒布局
- [ ] `rankings` prop 传入并正确渲染排名
- [ ] 在 `RankingsHubView.vue` 注册动态加载
- [ ] 在 `seasonPresets.js` 注册赛季预设元数据
- [ ] Playwright 验证规则展示和交互
