# 比赛系统 CRUD 规格文档

> 本文档定义轮次、比赛、局的增删改查完整逻辑，包括计分规则、状态机、边界情况。

---

## 1. 计分规则

### 1.1 局 (Game) 结束条件

标准 21 分制（业余简化版，不封顶 30 分）：

| 条件 | 结果 |
|------|------|
| 一方达到 21 分 **且** 领先 ≥ 2 分 | 该方获胜，局结束 |
| 双方 20:20 | 继续，直到一方领先 2 分 |
| 双方分数相等 | 不能结束本局 |
| 最高 30 分封顶 | 先到 30 分者获胜（无论分差） |

### 1.2 比赛 (Match) 结束条件

| 赛制 | 条件 |
|------|------|
| BO1（一局定胜负） | 1 局完成后结束 |
| BO3（三局两胜） | 一方先赢 2 局；或打完 3 局 |
| BO7（七局打满） | 7 局全部完成后，按赢局数判定 |

### 1.3 轮次 (Round) 结束条件

轮次内所有比赛状态均为 `completed` 时，轮次自动标记为 `completed`。

## 2. 状态机

### 2.1 比赛状态流转

```
                    ┌─ cancel → (恢复为 pending，删除已创建的局)
                    │
create → pending ───┤
                    │
                    └─ start → in_progress ──→ completed
                                   │               ↑
                                   │ 暂停(保持 in_progress)
                                   │               │
                                   └── 继续 ───────┘
```

### 2.2 局状态流转

```
match.start() → G1:in_progress, G2..Gn:pending

endGame(G1) → G1:completed, G2:in_progress
endGame(G2) → G2:completed, G3:in_progress (if BO3)
...
最后局结束 + match completed → 不激活新局
```

### 2.3 轮次状态流转

```
create → pending
  → (当第一个 match 开始时) → in_progress
  → (当所有 match completed 时) → completed
```

## 3. CRUD 操作规范

### 3.1 轮次 (Round)

| 操作 | 前置条件 | 行为 |
|------|---------|------|
| **创建** | 赛季存在，轮次号不重复 | 创建 pending 状态的空轮次 |
| **编辑** | 轮次存在 | 可修改 status, venueManagerId |
| **删除** | — | **级联删除**所有比赛及局。需二次确认 |

**创建下一轮逻辑**：
- 找出当前赛季已有的最大轮次号 N
- 新建轮次号 = N+1（不超过 totalRounds）
- 创建后自动生成 3 场标准对阵（根据参赛选手排列组合）
- 若为 S4 下篇（roundNo 5-7），仅生成 1 场组合赛（BO7）

### 3.2 比赛 (Match)

| 操作 | 前置条件 | 行为 |
|------|---------|------|
| **创建** | 轮次存在且未完成 | 创建 pending 比赛 |
| **编辑** | 比赛未完成 | 可修改 teamA/B, bestOf, date, venueId |
| **删除** | — | **级联删除**所有局。若为轮次中最后一场比赛，需检查轮次状态 |
| **开始** | status=pending | 创建对应数量的局（按 bestOf），首局激活为 in_progress，轮次变为 in_progress |
| **取消** | status=in_progress | 恢复为 pending，**删除所有已创建的局** |

**边界情况**：
- 编辑已完成比赛：仅允许修改 date, venueId（不能改队伍和赛制）
- 删除进行中比赛：询问确认后级联删除所有局

### 3.3 局 (Game)

| 操作 | 前置条件 | 行为 |
|------|---------|------|
| **自动创建** | 比赛开始 | 按 bestOf 创建对应数量局 |
| **更新比分** | 局状态=in_progress | 更新 scoreA/scoreB |
| **结束本局** | 局状态=in_progress，双方分数不等，且满足 21 分制规则 | 标记 completed，计算 winner，激活下一局或结束比赛 |
| **编辑已完成局比分** | 局状态=completed | 修改 scoreA/scoreB，重新计算 winner，重新判定比赛/轮次状态 |
| **撤回局** | 局状态=completed 且是最后一局 | 恢复为 in_progress，若比赛 completed 则恢复为 in_progress，清除 winner |

**撤回逻辑详述**：
1. 检查是否为最后一个 completed 局
2. 删除该局之后的所有局（含 pending 局）
3. 将该局状态改为 in_progress，清除 winner 和 completed_at
4. 若比赛已完成，恢复为 in_progress，清除 match.winner
5. 若轮次已完成，恢复为 in_progress

**编辑已完成局比分边界情况**：
- 修改后比分相等 → 拒绝
- 修改导致 winner 改变 → 允许，重新计算 match 大比分
- 修改导致 match winner 改变 → 允许，更新 match.winner
- 修改导致 match 从 completed 变回未完成 → 允许（如原 2:0，改为 1:1）
- 修改导致 round 状态变化 → 自动级联检查

## 4. 前端交互设计

### 4.1 赛季页 → 轮次管理

```
┌─ 轮次记录 ─────────────────────┐
│ TPC第四赛季  第 6/7 轮          │
│ [创建下一轮]                    │
│                                │
│ ▼ R6  已完成                   │
│   M1  赵沂/胡肖涛 vs 王铮昊/张逸骋  2:0  [✏][🗑]│
│   M2  赵沂/王铮昊 vs 胡肖涛/张逸骋  1:2  [✏][🗑]│
│   M3  ...                      │
│                                │
│ ▼ R5  已完成                   │
│   ...                          │
└────────────────────────────────┘
```

**创建下一轮**：
- 点击按钮 → 弹出 Sheet 确认
- 显示自动生成的对阵
- 用户可调整对阵（交换选手）
- 确认后创建轮次 + 所有比赛

### 4.2 比赛页 → 比赛管理

```
┌─ 比赛 ────────────────────────┐
│ [+ 发起友谊赛]                  │
│                                │
│ 🔴 焦点战                      │
│ ┌ 赵沂/胡肖涛 vs 王铮昊/张逸骋 1:0 ┐ │
│ │ Live · BO3 · TPC第四赛季 R4   │ │
│ │ [继续记分]               [✏][🗑]│ │
│ └──────────────────────────────┘ │
│                                │
│ 📅 日历                        │
│ ...                            │
│                                │
│ 📋 历史                        │
│ 05-14 [赛季] 赵沂/胡肖涛 3:4 ...  │
│        第四赛季                  │
└────────────────────────────────┘
```

### 4.3 比赛详情页 / 记分页

**已完成比赛**（MatchDetailView）：
- 显示大比分 + 各局详情
- [编辑比分] 按钮 → 弹出编辑弹窗
- [删除比赛] 按钮

**进行中比赛**（ScoringView）：
- 大号实时计分板
- 各局进度条
- [结束本局] / [暂停] / [撤销] 按钮
- 各局可点击修改
- 记分按钮需符合 21 分制验证

## 5. 边界情况清单

| # | 场景 | 预期行为 |
|---|------|---------|
| 1 | 删除轮次（含进行中比赛） | 需确认，级联删除所有比赛和局 |
| 2 | 编辑已完成比赛的队伍 | 拒绝（只能改日期/场地） |
| 3 | 结束局时比分相等 | 拒绝，提示"比分不能相等" |
| 4 | 结束局时一方 <21 分 | 拒绝，提示"需达到21分且领先2分" |
| 5 | 撤回最后一个局 | 局+比赛恢复为 in_progress |
| 6 | 撤回非最后一个局 | 拒绝（只能撤回最后一局） |
| 7 | 编辑已完成局使 match winner 改变 | 允许，自动更新 |
| 8 | 编辑已完成局使比赛从 2:0 变为 1:1 | 允许，match 从 completed → in_progress |
| 9 | 创建轮次时赛季已完成 | 拒绝 |
| 10 | 开始比赛时轮次已完成 | 拒绝 |
| 11 | 删除比赛后轮次状态 | 若剩余比赛全部 completed → 轮次 completed；否则 in_progress/pending |
| 12 | BO7 赛制下结束局 | 不判定比赛结束，打满 7 局后按总赢局判定 |
| 13 | 20:20 平分时结束局 | 拒绝（需领先 2 分） |
| 14 | 29:29 时结束局 | 允许（先到 30 封顶，谁说先到 30 都不行——必须等待 30）|

## 6. 实现优先级

| 优先级 | 功能 | 原因 |
|--------|------|------|
| P0 | 局：更新比分 + 结束本局（含 21 分制验证） | 记分核心 |
| P0 | 比赛：开始 + 状态流转 | 记分入口 |
| P1 | 轮次：创建下一轮 + 自动生成对阵 | 新赛季/新轮次 |
| P1 | 比赛/局：编辑已完成数据 | 纠错 |
| P2 | 比赛/轮次/局：删除 | 数据管理 |
| P2 | 撤回局 | 容错 |
